PATTERNS IDENTIFIED FROM 3 MIXIN SAMPLES:
==========================================

1. GuiGraphics API Migration
   - Pattern: import net.minecraft.client.gui.GuiGraphics → GuiGraphicsExtractor
   - Pattern: Parameter type GuiGraphics → GuiGraphicsExtractor in @Inject methods
   - Affected files: CreateWorldScreenLatitudeToggleMixin, InGameHudMixin, HandledScreenCompassToggleMixin
   - Method calls on context (ctx.fill, ctx.text) stay the same

2. Method Name Changes in @Inject Descriptors
   - Key insight: @Inject method descriptors reference Minecraft internal method names
   - Method descriptors appear to stay unchanged (e.g., render, renderMainHud, drawSlot)
   - These match against the obfuscated/internal Minecraft code, not our parameter types
   - Parameter type in descriptor uses internal class names (DrawContext, RenderTickCounter)
   - Actual Java parameter types in our @Inject method use public API names

3. Static Overlay Method Calls
   - In InGameHudMixin, calling EwSandstormOverlayHud.render(context, tickCounter)
   - These static methods accept GuiGraphicsExtractor (or will after migration)
   - No changes needed to the call site; parameter type change handles it

4. Handler vs. Error Type Analysis
   CreateWorldScreenLatitudeToggleMixin (line 104):
     @Inject(method = "render(Lnet/minecraft/client/gui/DrawContext;IIF)V", at = @At("TAIL"))
     private void latitude$render(GuiGraphics context, ...)

   Error pattern: GuiGraphics not found (import error propagates to parameter type)
   Method descriptor references DrawContext (internal) - no error yet
   Fix: Change parameter type to GuiGraphicsExtractor

   InGameHudMixin (line 36):
     @Inject(method = "render", at = @At("TAIL"))
     private void globe$renderOverlay(GuiGraphics context, ...)

   Simple method name + tail injection, no detailed descriptor
   Same fix: GuiGraphics → GuiGraphicsExtractor

   HandledScreenCompassToggleMixin (line 52):
     @Inject(method = "drawSlot", at = @At("TAIL"))
     private void globe$drawCompassDisabledX(GuiGraphics ctx, Slot slot, ...)

   Different target method (drawSlot), same GuiGraphics parameter type
   Fix: GuiGraphics → GuiGraphicsExtractor
