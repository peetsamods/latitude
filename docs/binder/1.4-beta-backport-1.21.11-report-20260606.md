## Scope landed (Stage 1 / Bucket 1 — worldgen logic only)

This is an **algorithm transplant**, not a cherry-pick: the 1.4 source (26.1.2, Mojang mappings) was authored on a
**province/moisture** worldgen architecture that **does not exist in the 1.3.0 line** for 1.21.11 (no `ProvinceAuthority`,
no `Province` enum, no `enforceWarmProvinceFamily`, no moisture classifier). The pure-logic 1.4 changes were therefore
re-grounded onto the target's existing Yarn (`RegistryEntry<Biome>`) worldgen and validated map-based.

| 1.4 axis | source commit | status in 1.21.11 | how |
|---|---|---|---|
| Art VI `pickFrom` coherent noise | `da1bc0dd` | **LANDED (verbatim logic)** | `pickFrom` argmax over N coherent `ValueNoise2D` fields (`FALLBACK_COHERENCE_BLOCKS=128`, `FALLBACK_PICK_SALT`), replacing the `Math.floorDiv`+`hash64` cell-grid pick that was present and identical in this baseline. |
| Polar ice_spikes cap | `3f8e0366` | **LANDED (adapted)** | `keepPolarIceSpike` coherent cap (`POLAR_ICE_KEEP_THRESHOLD=0.45`, scale 256) in both `sanitizeLandBiome` overloads; source-source ice_spikes over the cap → `snowy_plains`. The baseline's `BAND_POLAR` block was *manufacturing* ice_spikes (forced forest/taiga→ice_spikes); rewired to `snowy_plains` (+`!path.contains("snowy")` guard) — Yarn idiom `getKey().map(k->k.getValue().getPath())`. |
| Equator overhaul: badlands gate + desert thinning | `91ffa425` + `10e10ffd` | **LANDED (re-grounded, province-independent)** | `demoteEquatorialBadlands`/`demoteEquatorialDesert` predicates collapsed into `demoteEquatorialDryBiomes` (both overloads), wired at the END of both `applyFinalSavannaClimateClamp` overloads before the savanna-tier pass. Keys on biome family + latitude ramp + coherent `ValueNoise2D` keep-noise (`BADLANDS_LAT_RAMP_LOW/HIGH=10/18`, `DESERT_LAT_RAMP_HIGH=12`, `DESERT_EQUATOR_KEEP_FRAC=0.40`). The WARM_DRY-province gate from the source is dropped (no province system here); the Earth-analog intent (no equatorial badlands; thinned-but-present equatorial desert; demote→savanna) is preserved exactly. |
| Warm-band rebalance (province wet-bias + WARM_WET variety + tier coherence 64→160 + badlands sub-province widen + `chooseBadlandsVariant`) | `8c73beab` | **DEFERRED (architecturally out of reach)** | Built entirely on `ProvinceAuthority.classifyWarm` / `enforceWarmProvinceFamily` / `Province` / moisture model — none exist in the 1.3.0 line. Porting it requires transplanting the whole province subsystem (Bucket-2-scale). The equatorial Earth-analog goal it targeted is delivered here by the province-independent equator gates above. `FALLBACK_COHERENCE_BLOCKS=128` is adopted; the `TIER_COHERENCE_BLOCKS=160` tier-coherence constant has no analog (no `pickFromWeighted` coherence tier in this baseline's selection path). |
