## Scope landed (Stage 1 / Bucket 1 — worldgen logic only)

This is an **algorithm transplant**, not a cherry-pick: the 1.4 source (26.1.2, Mojang mappings) was authored on a
**province/moisture** worldgen architecture that **does not exist in the 1.3.0 line** (no `ProvinceAuthority`,
no `Province` enum, no `enforceWarmProvinceFamily`, no moisture classifier). The pure-logic 1.4 changes were therefore
re-grounded onto the target's existing Yarn (`RegistryEntry<Biome>`) worldgen and validated map-based.

**Key porting fact:** the `1.21.1` 1.3.0 baseline `LatitudeBiomes.java` is **byte-identical** to the `1.21.11`
baseline (`diff` = 0 lines). The 1.21.11 transplant (proven, branch `port/1.4.0-beta-1.21.11`) therefore applies
**verbatim** — no Yarn-idiom adaptation was needed for this file. `compileJava` confirms the API touchpoints
(`RegistryEntry`, `Registry`, biome key lookup) resolve identically under the 1.21.1 Yarn mappings.
