## 1. Snow line — start snow a little further down the slopes
- Currently snow reads as **capped only on the very mountain top**; want it to
  begin a bit lower down the slope for a more natural treeline→snowline gradient.
- **CAUTION:** prior work specifically fought snow/ice *creep into warmer
  latitudes*. There is likely guard/clamp code that suppresses snow, and naively
  lowering the snow line could re-introduce that regression.
  - Relevant prior art to review: `AlpineSurfaceMixin` (`ALPINE_ROCK_Y`,
    latitude-graded snow), tree-line guard (`TreeLineVegetationGuardMixin`,
    `TREE_LINE_Y`), and any latitude snow-clamp in the surface/province code.
  - Goal: lower the snow *onset elevation* without widening the *latitude band*
    where snow is permitted. Keep the warm-latitude suppression intact.
- Want: snow onset graded by both altitude AND latitude (lower onset toward poles,
  higher/none toward equator), feathered so it isn't a hard cap ring.

## 2. Bespoke world-creation screen — fine-tuning + redesign
- **Bug:** at higher GUI Scale everything gets squished/overlapping.
- **Proposed direction:** move to a **"book" UI** with turnable pages:
  - Page 1 — world **name + seed**
  - Page 2 — **planisphere**: pick latitude band to spawn into + world size
  - Page 3 — **options**: game mode (survival/creative), world type
    (Latitude / vanilla / vanilla superflat / vanilla amplified), bonus chest,
    starting compass, etc.
- **First step is layout brainstorm** — agree on the page breakdown + responsive
  behavior across GUI scales before building.
