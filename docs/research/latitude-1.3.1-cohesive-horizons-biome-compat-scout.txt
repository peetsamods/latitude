Latitude local policy findings
- Biome eligibility is two-part:
  1. Java authority in src/main/java/com/example/globe/world/LatitudeBiomes.java via globe:* TagKey<Biome> selectors such as globe:lat_temperate_primary, globe:lat_subpolar_primary, globe:lat_ocean_temperate, and river/ocean checks against vanilla BiomeTags.
  2. Data membership in src/main/resources/data/globe/tags/worldgen/biome/*.json.
- Current tag namespace is globe, not latitude.
- Current tag files use plain required string IDs only. Example: lat_temperate_primary.json contains raw minecraft:* values; no optional entries are present anywhere in the current globe biome-tag set.

Optional external biome ID safety
- Safe only if Latitude uses optional tag entries for modded biome IDs.
- Unsafe if absent-mod biome IDs are added as today's bare string entries, because current files encode required entries only.
- Evidence: Minecraft's tag loader exposes optional entries in TagEntry.createOptional / createOptionalTag and Fabric datagen exposes addOptional, so the engine has a native optional path; Latitude just is not using it today.
- Audit conclusion: native compatibility is technically feasible without hard requiring the biome mods, but only through optional-entry encoding in the Latitude tag JSONs.

First-wave recommendation
- Start with Oh The Biomes We've Gone, Nature's Spirit, and Biomes O' Plenty.
- Hold Terralith for the second scout slice unless the first implementation slice is limited to a very small optional-ID pilot.

Proposed compatibility policy shape
- Tropical: only source-tagged hot/wet jungle-rainforest-tropics biomes; no inland mangrove unless explicitly wetland/coastal.
- Subtropical: hot/dry savanna, chaparral, mediterranean, dry woodland, warm scrub.
- Temperate: plains, groves, deciduous forest, mixed forest, meadow, orchard, floral fields.
- Subpolar: boreal/taiga/cold conifer/cool highland and other source-tagged cold land biomes short of full ice.
- Polar: tundra, glacier, snowfield, icy wasteland, frozen barrens.
- Coastal required: beaches, shores, dunes, coasts, mangrove marshes only when source tags or docs mark them coastal.
- Mountain required: peaks, ridges, cliffs, crags, alpine/highland only when source tags or docs mark mountain/highland terrain.
- Wetland required: bayou, bog, marsh, swamp, wetlands only when source tags or docs mark wet/swamp.
- Ocean/river/frozen handling: keep Latitude's current vanilla river/ocean authority first; only add modded ocean/river/frozen biomes after a dedicated aquatic audit.
- Ignore/non-overworld: nether, end, cave-only, structure/helper pseudo-biomes, and any biome whose source classification is contradictory or unclear.

Blockers
- Latitude cannot safely reference absent mods with the current bare-string tag encoding; optional entry form is required.
- Terralith's license and datapack/mod hybrid status warrant a narrower follow-up before bulk inclusion.
- BOP helper tags are less cleanly standardized on Fabric than BYG/Nature's Spirit, so the first BOP pass should stay conservative.
