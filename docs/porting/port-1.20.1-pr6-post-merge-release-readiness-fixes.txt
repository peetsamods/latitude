## Fixes

- `src/main/resources/globe.mixins.json`
  - Changed Mixin `compatibilityLevel` from `JAVA_21` to `JAVA_17`.
  - Root cause: the Modrinth 1.20.1 profile launches Java 17, and Mixin rejected `JAVA_21` during bootstrap.
- `src/main/java/com/example/globe/mixin/ChunkRegionWarmSnowTrapMixin.java`
  - Removed production references to `com.example.globe.debug.WarmSnowTrapStats`.
  - Root cause: the release jar excludes `com/example/globe/debug/**`, but the production warm-snow setBlockState mixin referenced that excluded debug class during worldgen.
- `src/main/java/com/example/globe/dev/InvariantScan.java`
  - Updated the scan from the old downloading-terrain text invariant to the current LevelLoadingScreen overlay and client tick handoff invariant.
  - The dev package remains excluded from the runtime jar.
