OCEAN/RIVER CLIMATE AUTHORITY
------------------------------
  Both use:
    tBase = abs(blockZ) / effectiveRadius
    t     = applyBoundaryJitter(blockX, blockZ, effectiveRadius, tBase)
    band  = bandForAbsLatFraction(t)                    ← from JITTERED t
    bandIndex = bandIndexForBand(band)                  ← hard integer snap

  applyBoundaryJitter:
    amplitude:  clamp(radius * 0.03, 120, 700) blocks  → ~150 blocks at R=5000
    wavelength: clamp(radius * 0.45, 2600, 12000) blocks → 2600 blocks at R=5000
    Effect: long-wave (~2600 block) sinusoidal roughness on the boundary position.
    Within each wave segment, the boundary is still a single-pixel-precision hard cutoff.

MISMATCH CONFIRMED — RULER-STRAIGHT EDGES
------------------------------------------
  Ocean and frozen_river use bandIndex (jitter only, hard integer snap).
  Land uses latitudeBandIndexWithBlend (jitter + 1408-block stochastic blend).

  The contrast causes:
  - Ocean band edges: wavy at large scale (2600-block wavelength) but hard-cutoff
    at fine scale (within each wave segment, ruler-straight at pixel level).
  - Land band edges: pixel-level organic mixing in a 1408-block transition zone.
  - Visible artifact: ruler-straight temperature transitions at ocean/river band
    boundaries (tropical→subtropical ocean, subpolar→polar frozen_river) despite
    land nearby having blended edges.

PLANNED FIX
-----------
  Narrowest change: compute blendedBandIndex = latitudeBandIndexWithBlend(...)
  ONCE, after the beach early-return and before the river/ocean checks.
  Use it for river freeze threshold and ocean type selection.
  Also reuse it as landBandIndex (eliminating the duplicate call).
  Both pick() overloads (Registry and Collection) get the same fix.
