MISMATCH FOUND
--------------
  Location: tools/atlas/viewer/index.html  — function inferBiomeBandMask()
            tools/atlas/viewer.html         — function inferBiomeBandMask()

  Root cause: The JavaScript fallback function inferBiomeBandMask() incorrectly
  maps several biomes to TROPICAL-only when BiomeBandPolicy.java allows TROPICAL+SUBTROPICAL.
  This causes false-positive Band Violation flags in the viewer overlay for any run
  that lacks a valid legend.json, AND for biomes not present in legendMap.

  Specific mismatches (JS fallback vs Java canonical):
    Biome pattern   | JS fallback       | Java BiomeBandPolicy (canonical)
    ----------------+-------------------+---------------------------------
    jungle family   | tropical only     | tropical, subtropical
    mangrove        | tropical only     | tropical, subtropical
    desert/badlands | tropical only     | tropical, subtropical
    savanna family  | tropical only     | tropical, subtropical

  Note: The primary path (legendMap from legend.json) is CORRECT for new runs.
  The mismatch only fires for old runs (no legend.json) or biomes absent from BiomeBandPolicy.

PLANNED ACTION
--------------
  Narrowest fix: update inferBiomeBandMask() in both viewer files to match BiomeBandPolicy:
    jungle/mangrove  → ['tropical', 'subtropical']
    desert/badlands  → ['tropical', 'subtropical']
    savanna          → ['tropical', 'subtropical']
